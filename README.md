# Mini NPU Simulator

본 프로젝트는 3x3부터 25x25 크기의 패턴을 판별하는 NPU 시뮬레이터를 Python으로 구현한 과제입니다. MAC(Multiply-Accumulate) 연산을 통해 입력 패턴과 필터의 유사도를 계산하고, 최적의 필터를 판정합니다.

## 실행 방법

1.  **환경**: Python 3.8 이상 (외부 라이브러리 불필요)
2.  **실행**: `python main.py`
3.  **모드 선택**:
    *   `1`: 사용자 입력 모드. 3x3 크기의 필터 A, B와 패턴을 직접 행 단위로 입력합니다.
    *   `2`: `data.json` 분석 모드. 파일로부터 다양한 크기의 필터와 패턴을 로드하여 일괄 판정합니다.

## 구현 요약

*   **라벨 정규화**: 입력 데이터의 `+`, `cross`, `x` 등 다양한 라벨을 `Cross`와 `X`로 표준화하여 일관된 판정 로직을 적용했습니다.
*   **MAC 연산**: NumPy와 같은 외부 라이브러리 없이 중첩 반복문을 사용하여 위치별 곱셈 및 누적 합계를 직접 구현했습니다.
*   **동점 처리 (Epsilon)**: 부동소수점 오차를 고려하여 `abs(score_a - score_b) < 1e-9`인 경우 `UNDECIDED`로 판정하는 정책을 수립했습니다.
*   **데이터 검증**: 행/열 크기 불일치, 숫자 파싱 오류 등을 검증하며, `data.json` 분석 시 오류가 발생한 케이스만 `FAIL` 처리하고 전체 프로세스는 유지되도록 설계했습니다.

## 결과 리포트

### 1. 실패 원인 분석 (FAIL Case Analysis)
현재 `data.json` 테스트 결과 **0 FAIL**을 달성했습니다. 모든 케이스가 `PASS`가 된 주요 요인은 다음과 같습니다:
*   **라벨 정규화**: `expected` 값에 들어있는 `+` 기호를 코드 내부에서 `Cross`로 변환하고, 필터 키의 `cross` 또한 `Cross`로 매핑하여 판정 기준을 일치시켰습니다.
*   **부동소수점 허용 오차**: `1e-9` 크기의 epsilon을 적용하여, 미세한 연산 오차로 인해 발생할 수 있는 잘못된 판정이나 동점 상황에서의 불안정성을 제거했습니다.
*   **엄격한 크기 검증**: 패턴 키(`size_N_idx`)에서 추출한 `N`과 실제 데이터 크기를 비교하여, 데이터 스키마 오류를 사전에 차단했습니다. 만약 크기가 맞지 않는 데이터가 유입된다면 시스템 중단 없이 해당 케이스만 `FAIL`로 기록됩니다.

### 2. 시간 복잡도 및 성능 분석
성능 측정 결과, 패턴의 크기 `N`이 증가함에 따라 연산 시간이 급격히 증가하는 것을 확인했습니다.

| 크기 (NxN) | 연산 횟수 (N²) | 예상 시간 복잡도 |
| :--- | :--- | :--- |
| 3x3 | 9 | O(N²) |
| 5x5 | 25 | O(N²) |
| 13x13 | 169 | O(N²) |
| 25x25 | 625 | O(N²) |

**분석 근거:**
MAC 연산은 `N x N` 행렬의 모든 요소를 한 번씩 순회하며 곱셈과 덧셈을 수행하므로, 이론적 시간 복잡도는 **O(N²)**입니다. 측정 데이터에서 `N`이 5에서 25로 5배 증가할 때, 연산 횟수는 25회에서 625회로 **25배(5²)** 증가합니다. 실제 측정된 평균 연산 시간 또한 이 연산 횟수 증가 비율에 비례하여 늘어나는 양상을 보였으며, 이는 구현된 알고리즘이 `O(N²)`의 복잡도를 충실히 따르고 있음을 증명합니다.
